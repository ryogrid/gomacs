## Codebase Patterns
- Module name is `goomacs`, so term package imports as `goomacs/term`
- main.go uses tcell types: Screen, EventKey, EventResize, StyleDefault, Key* constants, ModAlt
- SetContent in tcell takes (x, y, ch, combining []rune, style) - our term.Screen drops combining runes param
- tcell uses pointer receivers for events (*tcell.EventKey) - our term package matches this pattern
- Terminal.in field (io.Reader) allows injecting test input for keyboard parsing tests
- PollEvent uses two channels: `posted` (re-queued) has priority over `events` (stdin/resize)
- Esc timeout is 50ms to distinguish bare Escape from Alt+key or ANSI sequences

## Progress: Pure Go Terminal Backend

## 2026-03-10 - US-001
- Defined Screen interface, Event interface, KeyEvent, ResizeEvent, Style, KeyCode constants, ModMask constants in term/screen.go
- Files changed: term/screen.go (new)
- **Learnings for future iterations:**
  - The interface is minimal, matching only what goomacs currently uses from tcell
  - Style uses a bitmask approach (uint8) for extensibility
  - Events use a marker method pattern (isEvent()) for the interface
  - KeyCode constants start at 256 to leave room for rune values
- TIOCGWINSZ (0x5413) returns winsize struct: Row, Col, X, Y (all uint16)
- SIGWINCH signal is used to detect terminal resize; use os/signal.Notify
- Terminal struct has an `events` channel (buffered, cap 64) for delivering events to PollEvent
- Terminal raw mode uses TCGETS (0x5401) / TCSETS (0x5402) ioctl on Linux
- Screen buffer uses [][]cell where cell has rune + Style; prev buffer used for diffing
- ANSI coordinates are 1-based: \033[row;colH to position cursor
- Reverse video: \033[7m to enable, \033[0m to reset
- Show() uses bufio.Writer for buffered output, flushed once per call
- Alternate screen buffer: \033[?1049h to enter, \033[?1049l to exit
- termios struct layout matches Linux kernel (Iflag, Oflag, Cflag, Lflag, Line, Cc[32], Ispeed, Ospeed)

## 2026-03-10 - US-004
- Implemented cell-based screen buffer with diff-based rendering using ANSI escape sequences
- Added cell type, makeBuffer helper, and screen buffer fields (cells, prev, width, height, cursorX, cursorY, out)
- Clear() resets all cells to space with default style
- SetContent() stores cell data, bounds-checked
- Show() diffs current vs previous buffer, writes only changed cells with ANSI cursor positioning
- Show() applies reverse video (\033[7m) and resets (\033[0m) per cell
- ShowCursor() stores cursor position, Show() positions hardware cursor and shows it
- Sync() marks all prev cells dirty to force full redraw
- Output buffered via bufio.Writer, flushed once per Show()
- resize() handles terminal size changes by reallocating buffers
- Files changed: term/terminal.go
- **Learnings for future iterations:**
  - Use sentinel value (ch = -1) in prev buffer to force initial draw
  - ANSI is 1-based so always +1 when converting from 0-based buffer indices
  - bufio.Writer wrapping os.Stdout is efficient for batching escape sequences
  - resize() must be called at the start of Show() to handle size changes gracefully

## 2026-03-10 - US-005
- Implemented keyboard input parsing in term/terminal.go
- PollEvent() blocks and returns parsed events; PostEvent() re-queues events with priority
- readInput() goroutine reads from t.in (io.Reader) and calls parseInput()
- parseInput() handles: control chars (0x00-0x1A, 0x1F, 0x7F), printable ASCII, multi-byte UTF-8
- parseEscSequence() handles: CSI arrow keys (\033[A/B/C/D), Alt+key (Esc + char)
- Bare Escape detected via 50ms timeout when Esc byte arrives alone
- Added `posted` channel for re-queued events, `in` field for injectable input reader
- Files changed: term/terminal.go
- **Learnings for future iterations:**
  - PollEvent priority: check `posted` channel first (non-blocking), then select on both
  - Esc timeout uses goroutine + select with time.After; leftover bytes from goroutine are re-parsed
  - Control char mapping: 0x01=C-a, 0x09=Tab, 0x0d=Enter, 0x1f=C-_, 0x7f=Backspace2
  - The `in` field defaults to os.Stdin but can be set to bytes.Reader for testing
