## Codebase Patterns
- Module name is `goomacs`, so term package imports as `goomacs/term`
- main.go uses term.Screen interface with term.KeyEvent, term.ResizeEvent, term.Style types
- Buffer struct holds Lines, CursorR/C, ScrollOffset, Modified, Filename, KillRing, Mark, UndoStack
- Event loop in main.go: PollEvent → dispatch by mode (search/C-x prefix/normal) → redraw
- cell struct in terminal.go: {ch rune, style Style} — used for diff-based rendering in Show()
- Style is now a struct (fg Color, bg Color, reverse bool, bold bool); StyleDefault is a var, not const
- writeStyledCell() on Terminal emits combined ANSI sequence
- Highlighter in highlight.go: NewHighlighter(filename) → nil if no lexer; Highlight(lines) caches [][]term.Style; StyleAt(row,col) returns cached style
- drawWindowContent(screen, win, sh) renders buffer content in window's row range
- Tab expansion handled in rendering (bufColToVisualCol)
- `buf` is a local alias for `buffers[activeBufferIdx]`
- Window struct: Buffer, ScrollOffset, StartRow, Height; ViewHeight() = Height-1
- recalcWindows(windows, screenHeight) distributes height evenly, reserves 1 row for message line
- C-x 2 splits vertically (top/bottom), C-x 3 splits horizontally (side-by-side), C-x o cycles windows, C-x 0 closes active, C-x 1 keeps only active

- e2e/ directory contains E2E tests; harness.go provides Harness struct for tmux-based testing
- Harness.Start() creates detached tmux session, Harness.SendKeys() sends keys, Harness.Capture() returns screen rows
- Session names are sanitized (/ . : replaced with -) and have random suffix for parallel test support
