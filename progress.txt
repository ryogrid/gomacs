## Codebase Patterns
- Buffer is stored as `[][]rune` (Lines field), one slice per line
- Buffer tracks cursor via `CursorR`/`CursorC` fields
- Cursor movement methods on Buffer: MoveForward, MoveBackward, MoveUp, MoveDown, MoveBeginningOfLine, MoveEndOfLine
- Use `screen.ShowCursor(col, row)` to position the terminal cursor
- `NewBuffer()` creates empty buffer, `NewBufferFromFile(filename)` loads from disk
- `drawBuffer(screen, buf)` renders buffer to tcell screen, must call `screen.Show()` after
- Long lines truncated at screen width, lines beyond screen height not rendered
- Use `os.ReadFile` + `strings.Split` for file loading, trim trailing newline first
- Buffer tracks `ScrollOffset` (top visible line); use `AdjustScroll(viewHeight)` after cursor changes
- Cursor display uses `screen.ShowCursor(buf.CursorC, buf.CursorR - buf.ScrollOffset)` to account for scroll
- `drawBuffer` renders from `buf.ScrollOffset`, mapping buffer row to screen row via `bufRow = screenRow + ScrollOffset`
- Alt key bindings use `ev.Modifiers()&tcell.ModAlt != 0` with `ev.Rune()` check; also handle Esc-prefix for terminals
- Handle both `tcell.KeyBackspace` and `tcell.KeyBackspace2` for cross-terminal compatibility
- Regular character insertion in KeyRune case goes in the `else` branch after the Alt modifier check
- Status line on row `height-2`, message area on row `height-1`; text area height = `textAreaHeight(screenHeight)` = `height - 2`
- Use `tcell.StyleDefault.Reverse(true)` for status line reverse video styling
- C-x prefix key: set `prefixCx = true`, show "C-x-" in message area, then handle second key on next event
- Message area is on row `height-1`; drawn by `drawMessageLine(screen, msg)` after `drawBuffer`
- Buffer.Save() writes to temp file then os.Rename for safe writes; clears Modified on success
- `errNoFilename` is a package-level sentinel error for buffers with no filename
- For "repeat to confirm" patterns (like force-quit), use a warning flag that resets on non-related keypresses
