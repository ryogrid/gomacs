## Codebase Patterns
- Module name is `goomacs`, so term package imports as `goomacs/term`
- main.go uses term.Screen interface with term.KeyEvent, term.ResizeEvent, term.Style types
- Buffer struct holds Lines, CursorR/C, ScrollOffset, Modified, Filename, KillRing, Mark, UndoStack
- Event loop in main.go: PollEvent → dispatch by mode (search/C-x prefix/normal) → redraw
- cell struct in terminal.go: {ch rune, style Style} — used for diff-based rendering in Show()
- Style is now a struct (fg Color, bg Color, reverse bool, bold bool); StyleDefault is a var, not const
- writeStyledCell() on Terminal emits combined ANSI sequence
- Highlighter in highlight.go: NewHighlighter(filename) → nil if no lexer; Highlight(lines) caches [][]term.Style; StyleAt(row,col) returns cached style
- drawWindowContent(screen, win, sh) renders buffer content in window's row range
- Tab expansion handled in rendering (bufColToVisualCol)
- `buf` is a local alias for `buffers[activeBufferIdx]`
- Window struct: Buffer, ScrollOffset, StartRow, Height; ViewHeight() = Height-1
- recalcWindows(windows, screenHeight) distributes height evenly, reserves 1 row for message line
- C-x 2 splits vertically (top/bottom), C-x 3 splits horizontally (side-by-side), C-x o cycles windows, C-x 0 closes active, C-x 1 keeps only active
- Minibuffer: minibufferMode, minibufferPrompt, minibufferInput, minibufferCallback are **package-level vars** in main.go — Tab handler checks prompt to decide completion type
- Alt keys dispatched at ~line 983 in main.go: M-v, M-w, M-<, M->
- regionBounds() returns (startR, startC, endR, endC); SaveUndo() snapshots for single-step undo
- detectCommentStyle(filename, content) returns CommentStyle; check LinePrefix != "" for line comments, else use BlockStart/BlockEnd
- Command struct uses func(*Buffer, *string) signature; RegisterCommand in init(); FindCommand/FindCommandsByPrefix for lookup
- CaptureWithEscapes() on Harness captures tmux pane with ANSI escapes preserved (-e flag)
- e2e/ directory contains E2E tests; harness.go provides Harness struct for tmux-based testing
- editorScreen (package-level in grep.go) allows goroutines to PostEvent to wake PollEvent
- grepResultCh channel + non-blocking select after event switch handles async grep results
- executeGrepCommand(cmd) launches goroutine, sends on grepResultCh, posts synthetic event
- buffers, activeBufferIdx, previousBufferIdx are **package-level vars** — mode handlers can switch/remove buffers
- After mode handler returns true, main.go auto-refreshes `buf` and `activeWin.Buffer` from activeBufferIdx
- Minibuffer only supports Backspace for editing (no C-a, C-k, cursor movement) — in e2e tests, use SendKeysRepeat("BSpace", N) to clear pre-populated input
- StartInDir(t, dir, binary, args...) launches tmux session in a specific directory (uses tmux -c flag)
- grep exit code 1 (no matches) with no stderr → "No matches found" message (not an error)
- For Alt keys in e2e tests, use tmux `M-n`/`M-p` syntax directly — do NOT use `Escape` + sleep + `n` (terminal library may not combine them)
- grep -rnH output order is not alphabetical — pipe through `| sort` for predictable test assertions
